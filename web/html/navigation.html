二、判断VPS服务器的IP是否“送中”及解决方法

  1. 判断方法：
  2. 点击打开 https://music.youtube.com/，能正常访问，代表没“送中”，反之就是“送中”了。
  3. 如果送中了如何解决去【拉回来】？解决方法：
  4. 关闭/取消登录了谷歌账户的APP定位权限/授权；
  5. 将常用的一个谷歌账号的位置记录功能打开；
  6. 打开谷歌浏览器，登录开了位置记录功能的谷歌账号，并安装 Location Guard 扩展插件；
  7. 打开Location Guard插件，选择Fixed Location，并在给出的地图上单击，即可标记上你想要IP所处的国家/地区 Google IP定位错误，使用Location Guard修改；
  8. 转换到Options选项，Default level默认设置为Use fixed location；
  9. 访问 谷歌地图服务 获取当前GPS位置，确认是否已修改成功；
  10. 谷歌搜索my ip，即可看到谷歌IP定位到了刚才地图上标记的位置；
  11. 最后，通过 此网页向谷歌报告IP问题。

三、在VPS服务器部署“订阅转换”功能

  1. 操作步骤：：
  2. 进入脚本输入x-ui命令调取面板，选择第【25】选项安装订阅转换模块；
  3. 等待安装【订阅转换】成功之后，访问地址：https://你的域名:15268 ；
  4. 因为在转换过程中需要调取后端API，所以请确保端口 8000 和 15268 是打开放行的；
  5. 直接复制脚本中提供的【登录地址】，进入后台，点击【节点列表】----->>>【添加节点】；
  6. 接下来点击左边侧边栏的【订阅列表】去【添加订阅】；
  7. 最后一步，点击【客户端】，即可导入Clash等软件中使用；
  8. 此功能集成到〔x-panel-ce〕中，是为了保证安全，不会造成链接泄露。

四、如何保护自己的IP不被封锁被墙掉？

  1. 使用安全代理协议：： 加密是必备，推荐使用vless+reality+vision协议组合；
  2. 避免共享节点：： 尽量不要在不同的地区，多个省份之间不要共同连接同一个IP；
  3. 隔离IP和端口：： 避免多个用户或设备在不同地理位置漫游时使用同一个IP和端口，要分开；
  4. 控制流量：： 不要在单台VPS上长时间进行大流量下载，在一天内不要流量过高，适时切换节点；
  5. 使用高位端口：： 创建入站时，尽量使用 40000 至 65000 之间的高位端口号。
  6. 核心总结：： 不要多终端、多省份、多朋友共同使用同一个IP和端口。多创建几个入站，各用各的，避免被 GFW 识别为机场特征。 使用〔x-panel-ce〕多创建几个【入站】， 多做几条备用，各用各的！各行其道才比较安全！ GFW的思维模式是干掉机场，机场的特征个人用户不要去沾染，自然IP就保护好了。

一、关于 x-panel-ce

  1. 项目性质： 这是 xeefei/X-Panel 的 GPL-3.0 开源 fork，命名为 x-panel-ce（Community Edition）。
  2. 无收款 / 无授权码： 本仓库不实现任何"授权码"、"会员等级"、"积分"、"购买机器人"相关代码路径。所有上游 Pro 功能将以 OSS 方式重写或中性化，目标是社区可自由使用、修改、分发。

三、用户隐私底线

x-panel-ce 在 Stage 0.1 紧急安全清理 中移除了上游硬编码的第三方 Telegram Bot Token 和 chat ID。该机制原本会在以下三个时机将主机名 / TG 用户名 / TG 用户 ID 异步上报到上游开发者控制的频道，且部署用户毫无知情或退出途径：

  1. 每次 SendReport() 执行（"心跳报告"）；
  2. 抽奖回调中奖时（"中奖报告"）；
  3. 抽奖回调未中奖时（"未中奖报告"）。

注：抽奖功能本身（含 runLotteryDraw / sendLotteryGameInvitation / 三处 callback / 主菜单"🎁 娱乐抽奖"按钮 / LotteryWin 数据表 / ceReportPrefs.Lottery 字段）已在后续清理中整段移除，与 CE 开源、自托管、无收款定位无关。

移除细节与 commit hash 详见仓库 NOTICE.md 第 4 节第 5 项。
